1 Web Analizi web-analyzer 
web-analyzer subagent'ını kullan.

[URL] adresini analiz et. Tüm 6 analiz katmanını eksiksiz tamamla:
- Sayfa yapısı ve route'lar
- UI bileşenleri (her sayfada)
- API endpoint'leri (URL, method, request/response)
- Veri modeli (entity'ler ve ilişkiler)
- İş mantığı ve kullanıcı akışları
- Teknoloji stack (frontend / backend / 3rd party)

Çıktıyı ~/ai-dev-team/analysis/web-analysis-report.html olarak kaydet.

Tamamlandığında raporla:
- Toplam sayfa sayısı
- Tespit edilen API endpoint sayısı
- Veri modeli sayısı
- Kullanıcı akışı sayısı
- Tespit edilen risk sayısı
1.5 Marka Kimliği Analizi brand-analyzer 
brand-analyzer subagent'ını kullan.

Şu dosyayı oku:
1. ~/ai-dev-team/analysis/web-analysis-report.html

Ayrıca [URL] adresine WebFetch ile eriş ve CSS dosyalarını analiz et.

Marka kimliğini analiz et:
- Renk paleti (primary, secondary, accent, background, text, status renkleri)
- Tipografi (font ailesi, ağırlıklar, boyut skalası)
- Spacing ve grid sistemi
- Border radius ve shadow değerleri
- İkon seti ve stili
- Buton ve form stilleri

Çıktıları kaydet:
- ~/ai-dev-team/docs/brand-tokens.json
- ~/ai-dev-team/docs/brand-style-guide.html

Özet: ana renk, tipografi, ikon kütüphanesi, güven skoru (CSS'den mi / tahmini mi?)
2 Mimari Tasarım system-architect 
system-architect subagent'ını kullan.

~/ai-dev-team/analysis/web-analysis-report.html dosyasını oku.

5 karar alanını tamamla:
1. Backend teknoloji seçimi (framework, DB, ORM, auth, hosting)
2. API tasarımı (REST vs GraphQL, endpoint yapısı, error standardı)
3. Veritabanı şeması (tablolar, ilişkiler, index stratejisi)
4. iOS mimari kararları (pattern, networking, state, persistence, navigation)
5. Proje yapısı (backend + iOS dizin yapıları)

Her karar için: seçim, gerekçe, riskler, alternatifler.

Çıktıyı ~/ai-dev-team/docs/architecture-decisions.md olarak kaydet.
3 BA Doküman Üretimi ba-doc-generator

Şu iki dosyayı oku:
1. ~/ai-dev-team/analysis/web-analysis-report.html
2. ~/ai-dev-team/docs/architecture-decisions.md

Kapsamlı iş analizi dokümanı üret.

Kurallar:

- Her modül: açıklama, iş akışı diyagramı (mermaid), FR, BR, AC, VAL
- GIVEN-WHEN-THEN formatı KULLANMA
- Her gereksinim somut ve test edilebilir olmalı

Çıktıyı ~/ai-dev-team/docs/ba-document.md olarak kaydet.
