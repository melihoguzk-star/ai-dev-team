Genel Bakış

Site amacı, hedef kitle ve temel özellikler

JSONPlaceholder, geliştiricilerin prototipler ve test senaryoları için kullanabileceği ücretsiz bir sahte (fake) REST API servisidir. typicode tarafından geliştirilmiştir.

Hedef Kitle: Frontend geliştiriciler, mobil geliştiriciler, API test yapanlar, eğitim amaçlı kullananlar.

Temel Özellikler:

  • 6 farklı kaynak tipi: Posts, Comments, Albums, Photos, Todos, Users
  • Tam CRUD desteği: GET, POST, PUT, PATCH, DELETE
  • Nested route'lar (1 seviye)
  • Query parametreleriyle filtreleme
  • Sabırtık response — yazma işlemleri simüle edilir, gerçekte değişmez
  • Authentication yok — tamamen açık erişim

İş Mantığı ve Kullanıcı Akışları

3 ana akış, iş kuralları

Kullanıcı Akışları

flowchart LR A["Kullanıcı Seçimi"] --> B["Kullanıcı Profili"] B --> C["Posts Listesi"] B --> D["Albums Listesi"] B --> E["Todos Listesi"] C --> F["Post Detay"] F --> G["Yorumlar"] D --> H["Album Detay"] H --> I["Fotoğraflar"] style A fill:#0D9488,stroke:#0F766E,color:#fff style B fill:#F0FDFA,stroke:#99F6E4

İş Kuralları

  1. Veri Her gönderi (post) bir kullanıcıya (userId) aittir — 1:N ilişki
  2. Veri Her yorum (comment) bir gönderiye (postId) aittir — 1:N ilişki
  3. Veri Her albüm bir kullanıcıya aittir, her fotoğraf bir albüme — zincirleme 1:N
  4. Davranış Yazma işlemleri (POST/PUT/PATCH/DELETE) simüledir — sunucu gerçek veriyi değiştirmez
  5. Davranış POST isteği her zaman sabit id döner (kaynak uzunluğu + 1)
  6. Filtreleme Query parametreleriyle filtreleme: ?userId=1, ?postId=1 vb.
  7. Filtreleme Nested route'lar filtrelemeye eşdeğer: /posts/1/comments = /comments?postId=1
  8. Auth Authentication yok — tüm endpoint'ler açık
  9. Auth Rate limiting yok (tahmin — dokümante edilmemiş)
  10. Auth Pagination desteği yok — tüm veriler tek seferde döner
